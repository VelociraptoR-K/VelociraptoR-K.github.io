# Регрессия с применением Scikit-Learn

## Готовый ipynb-борд с заполненными ячейками по ссылке: [scikit-learn_regression](https://colab.research.google.com/drive/1FK1ncUXFrbEhlFYGYg8DU3lRIYM1M4KT?usp=sharing)

### Пошаговый алгоритм интеграции обученной scikit-learn модели с веб-сервисом на Django:

**Этап 0. Сохранение артефактов обучения**

1.  **Зафиксировать модель.** Обученную модель нужно сохранить на диск в виде файла, используя инструменты сериализации Python. Это позволяет отделить этап экспериментов в Jupyter от этапа промышленного использования в веб-сервисе.
    
2.  **Сохранить «предобработчик».** Если перед обучением данные стандартизировались или масштабировались, объект-масштабировщик также необходимо сохранить в отдельный файл. Без него модель не сможет правильно воспринять новые данные с веб-формы.
    

**Этап 1. Создание Django-проекта и приложения**  
3. **Инициализировать проект.** Создаётся стандартная структура каталогов для веб-фреймворка Django.  
4. **Выделить функционал.** Внутри проекта создаётся отдельное приложение, которое будет отвечать за всё, что связано с предсказаниями. Это делается для чистоты кода.  
5. **Прописать регистрацию.** Новое приложение нужно обязательно зарегистрировать в файле глобальных настроек проекта.

**Этап 2. Организация хранения и загрузки модели**  
6. **Создать хранилище на сервере.** В папке нового приложения создаётся специальная директория, куда физически складываются сохранённые ранее файлы с моделью и масштабировщиком.  
7. **Написать загрузчик.** Реализуется скрипт, задача которого — при старте веб-сервера один раз прочитать файлы с диска и загрузить модель в оперативную память. Это критически важно для производительности, чтобы не читать файл заново при каждом клике пользователя.

**Этап 3. Создание пользовательского интерфейса**  
8. **Описать веб-форму.** Создаётся класс формы, где перечисляются все поля, которые пользователь должен заполнить. Каждое поле соответствует одному из входных признаков модели (в нашем случае их двадцать). Здесь же настраиваются подписи и типы данных (целые или дробные числа).  
9. **Создать обработчик запросов (View).** Пишется функция-контроллер, которая умеет:

*   При обычном заходе на страницу — показывать пользователю пустую форму.
    
*   При нажатии кнопки «Отправить» — извлекать введённые цифры из формы.
    

10.  **Собрать данные в правильном порядке.** В контроллере данные из полей выстраиваются строго в той последовательности, в которой модель видела их при обучении. Любая перестановка столбцов приведёт к неверному результату.
    
11.  **Применить масштабирование.** Собранный массив чисел пропускается через загруженный ранее масштабировщик, чтобы привести их к тому же диапазону, что и обучающая выборка.
    
12.  **Запустить предсказание.** Обработанные данные подаются на вход загруженной модели. На выходе получается число — результат регрессии.
    
13.  **Вернуть результат.** Контроллер отрисовывает ту же самую страницу с формой, но добавляет на неё блок с вычисленным предсказанием.
    

**Этап 4. Настройка маршрутизациии**  
14. **Связать адрес и логику.** В файле маршрутов приложения указывается, какой URL-адрес (например, главная страница сайта) вызывает написанный нами контроллер.  
15. **Подключить к проекту.** Файл маршрутов приложения регистрируется в головном файле маршрутов всего проекта, чтобы веб-сервер знал о его существовании.

**Этап 5. Запуск и демонстрация**  
16. **Создать HTML-шаблон.** Верстается простая веб-страница, где в центре находится форма ввода, а внизу — место для вывода результата.  
17. **Запустить веб-сервер.** Выполняется стандартная команда запуска Django из терминала.

### Итоговый процесс (что видит пользователь)

Пользователь открывает сайт, видит поля для ввода признаков, вводит их, нажимает кнопку — и система показывает вычисленное целевое значение. Модель работает на серверной стороне, пользователю не нужно устанавливать Python или знать о машинном обучении.
